filters

grayscale

The image was turned gray by looking at each individual pixel's red, green, and blue values and then averaging them. The pixel's red, green, and blue values were each set to this new value.

rainbow

The first step was to find average of each pixel's red, green, and blue value. Each pixel was then set to a value based on its location in the image as well as it's average RGB value.

An added feature to try later is an input control to flip the stripes from horizontal to vertical.

blur

This is was supposed to be a blurred image but it ended up looking like tv static and I like it.

red

Each pixel's average RGB value was first computed. If the average was smaller than 128 (half the color range) the red was set to double the previous amount and green and blue were each set to 0. If the average was larger than 128, the red was set to 255 (the maximum possible) and green and blue were each set to double the average minus 255.

It would be cool to have color sliders to manipulate the colors here so that a user could add different tints to the image.

hue shift

To achieve the effect here the CSS filter "hue rotate" was used. The colors of the image are rotated around the color wheel by a specified number of degrees. 0 degrees represents the original image, while 180 degrees is directly across the color wheel. This image is set to 180 degrees and looks really cool!

opacity

This is another built-in CSS filter. The opacity is set to 30%. If I did this by hand I would adjust the alpha value of the RGBA setting and maybe set it to different opacities based on the color values.

inverted rainbow stripes

This filter is similar to the rainbow filter above except it uses inverted colors and repeats color sections. Depending on the pixel's location in the image, either the maximum (255) red, green, or blue is subtracted from the pixel's actual red, green, or blue.
